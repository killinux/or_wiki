# NLP4LP:一个随 OptiMUS 提出的 (MI)LP 建模评测基准

> 一句话:NLP4LP 是一个用于评测「从自然语言端到端建模并求解」(MI)LP 能力的数据集,随 OptiMUS 系列工作发布;它刻意采用更长的问题描述并附带数据文件,以降低数据泄漏、贴近真实建模场景,值得收作 LLM×运筹评测的核心基准之一。

## 关键内容

- 初版规模为 67 个实例(54 个 LP + 13 个 MILP),每个实例配有专家给出的形式化、参考解,以及用于校验最优性的代码,使评测可以自动判定对错。
- 扩展版本随 OptiMUS-0.3 增长到约 355 个问题;部分综述在做标准化切分后引用的实例数约为 269 个(口径不同导致数字差异)。
- 评测口径是 solve / execution 准确率:只有当模型生成的优化代码能成功执行、且给出正确的最优目标值时,该实例才算「解出」——而不是只看是否产出了形式化标注。
- 相比 NL4Opt,NLP4LP 的题面描述更长、更复杂,覆盖设施选址 (facility location)、网络流 (network flow)、调度 (scheduling)、组合投资 (portfolio)、能源 (energy) 等多类应用场景。
- 数据集以 CC BY-NC 4.0 许可发布;数据与代码托管在 github.com/teshnizi/OptiMUS 及 Hugging Face。
- 发布脉络绑定 OptiMUS 系列论文(arXiv:2310.06116 / 2402.10172 / 2407.19633),作者为 Ali AhmadiTeshnizi、Wenzhi Gao、Madeleine Udell。
