# MAMO：带求解器的数学建模基准 / MAMO: a Mathematical Modeling Benchmark with Solvers

> 一句话：MAMO 把评测焦点从"最终答案"转向"建模过程"——用求解器执行并验证 LLM 产出的模型，从而暴露简单建模与复杂建模之间的能力鸿沟。

## 关键内容

- **核心动机与规模**：现有数学基准只看 final answer 是否正确，MAMO 转而评测 modeling process（用求解器执行并验证生成的模型）。论文共收集 **1209 个问题**，覆盖常微分方程（ODE）与优化问题（LP/MILP）。（已核实：标题、5 位作者、year 2024、1209 题与 ODE+优化覆盖，来自 arXiv abstract / Semantic Scholar / GitHub 引用。）
- **任务划分（已核实计数）**：分两大类——ODE 共约 346 题（first-order 196、second-order 110、systems 40）；优化共 **863 题 = Easy LP 652 + Complex LP 211**。Easy LP 偏高中难度、Complex LP 为本科难度且混合 LP/MILP。（计数由多个独立来源与原文 PDF 拆分一致确认。）
- **评测方式（process evaluation，已核实）**：优化任务要求模型用 **.lp 格式**表达数学模型，再交由 **COPT（Cardinal Optimizer）** 求解器求最优值并与标准答案比对（求解器可替换）；ODE 任务执行模型生成的 Python 代码做数值验证。（.lp + COPT 来自官方 GitHub README 直接确认。）
- **主要结论（已核实，v2/v3 表格）**：GPT-4o 为最强模型，在带 LLM 语法/.lp 纠错（论文中标 ††）设定下：**ODE 一阶 68.37%、二阶 37.27%、systems 40.00%、Easy LP 87.42%、Complex LP 29.38%**；不带纠错时一阶/二阶为 67.86%/36.36%。整体表现"Easy LP 与一阶 ODE 高、Complex LP 与二阶 ODE/systems 低"，印证简单与复杂建模的鸿沟。*注：源 JSON 给出的 68.37/37.27/87.42 即对应此 GPT-4o †† 行；v1 旧表口径不同（如二阶 74.55%、Complex LP 32.23%），引用时需认准版本。*
- **影响与资源**：被 **ORLM、LLMOPT、OptMATH** 等后续工作复用为评测基准（如 OptMATH-Qwen2.5-32B 报告 MAMO ComplexLP 54.1%）；代码与数据见 `github.com/FreedomIntelligence/Mamo`。收录于 **NAACL 2025 Findings**（aclanthology: 2025.findings-naacl.394），arXiv 预印本编号 2405.13144；v3 题名改为 "LLMs for Mathematical Modeling: Towards Bridging the Gap between Natural and Mathematical Languages"。
