# 把 LLM 当优化器：OPRO（Large Language Models as Optimizers）

> 一句话：OPRO 提出用 LLM 本身充当优化器——在 meta-prompt 里描述任务并附上历史(解, 分数)轨迹，让模型迭代生成更优候选解；它是 "LLM-as-optimizer" 范式的代表作，尤其把 prompt optimization 做成了一个可搜索的优化问题。

## 关键内容

- **核心机制（无梯度 / 黑箱优化）**：把待解问题用自然语言写进 meta-prompt，并把此前评估过的(解, 目标值)按分数排序作为优化轨迹一并喂给 LLM；LLM 据此提出新一批候选解，评估后再回填进 meta-prompt，循环迭代。整个过程不需要梯度，只依赖对目标函数的黑箱打分。
- **热身演示（warm-up）**：先在两类小规模数学优化上验证可行性——线性回归（拟合连续参数）与小规模旅行商问题（TSP，组合优化），说明 LLM 能在轨迹引导下逐步逼近更优解。
- **主战场 = prompt optimization**：把"找最好的提示词"建模成优化问题，让 LLM 生成并改进指令。相比人工设计的 prompt，在 GSM8K 上最高提升约 **8%**，在 Big-Bench Hard (BBH) 的若干任务上最高提升达 **50%**。
- **标志性结果**：在 PaLM-2-L 上，OPRO 优化出的指令把 GSM8K 零样本准确率从 **71.8% 提升到 80.2%**；过程中"发现"了著名的提示句 **"Take a deep breath and work on this problem step by step."**。
- **适用边界**：方法适合**小规模 / 黑箱**场景；当搜索空间显著增大时性能退化（LLM 难以在大空间内稳定逼近最优）。代码开源：`github.com/google-deepmind/opro`。
- **出处**：ICLR 2024；arXiv:2309.03409，Google DeepMind。
