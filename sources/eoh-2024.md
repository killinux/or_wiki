# Evolution of Heuristics (EoH)：用 LLM 高效自动设计算法/启发式

> 一句话：EoH 把启发式表示为成对的自然语言 "thought" 与可执行代码并让二者协同进化（co-evolution），是自动启发式设计（AHD / automatic heuristic design）的演化范式代表作，在在线装箱上用远少的 LLM 查询预算就超过了 FunSearch。

## 关键内容

- 提出 Evolution of Heuristics (EoH) 框架：把一个启发式的"想法"同时表示为语言形式（human-like thoughts）与代码形式（executable code），结合 LLM 与 Evolutionary Computation 进行 AHD。区别于 FunSearch 仅在代码空间搜索，EoH 让 thoughts 与 codes **协同进化**。
- 核心机制是用 LLM 充当变异/交叉算子，在种群上迭代生成新的 thought+code 对，再用问题目标对生成的启发式做评估与选择，形成 thought 与 code 的 coevolution。
- 实验在常用组合优化基准上优于手工设计的 best-known 启发式以及近期 AHD 方法（含 FunSearch）。
- 在线装箱（online bin packing）问题上，在广泛研究的 Weibull instances 上，EoH 平均使用的箱子数显著少于人工最优启发式和 FunSearch 的启发式，且**所需 LLM 查询次数远更少**（更高的查询效率/预算节省是其主打卖点）。
- 发表于 ICML 2024（arXiv:2401.02051，2024-01-04 首次提交）；开源仓库 FeiLiu36/EoH。
- 后续衍生工作：MEoH（多目标版本，arXiv:2409.16867）与 EoH-S（arXiv:2508.03082），同属 LLM4AD（LLM for Algorithm Design）生态。
