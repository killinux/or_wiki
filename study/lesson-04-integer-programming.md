# 第 4 课:整数规划(IP / MILP)与分支定界

承接 [[lesson-03-duality]]。现实里「种 4.5 公顷」可以,但「派 2.3 个工人」「开 1.7 家店」不行。
当变量被强制为整数,问题会突然变难。

## 1. 为什么整数让问题变难

LP 里可行域是连续的多边形,顶点定理保证最优解在顶点上 → 单纯形能高效求解。
**整数规划的可行域是一堆离散的点**,没有「顶点」可爬,顶点定理失效。穷举所有整数点又会爆炸。

> **MILP**(混合整数线性规划)= 一部分变量要整数、一部分可连续。最常见的现实模型。

## 2. 致命陷阱:不能简单「四舍五入」

先解掉整数要求得到的 LP,叫 **LP 松弛(relaxation)**。很多人以为把它的小数解四舍五入就行 —— **大错**。

例子:
```
max  8x + 5y
s.t.  x +  y ≤ 6
     9x + 5y ≤ 45
     x, y ≥ 0 且为整数
```
LP 松弛最优:`x=3.75, y=2.25, z=41.25`。

- 直接舍入 (3,2):z=34,离最优很远。
- 舍入 (4,2):`9·4+5·2=46 > 45`,**根本不可行**!
- 真正的整数最优是 **(5,0),z=40** —— 你**猜都猜不到**它在 LP 解附近以外。

所以需要一套系统方法:**分支定界**。

## 3. 分支定界(Branch and Bound)

核心:**分而治之 + 用 LP 松弛当「上界」剪枝**。

- **松弛给上界(bound)**:整数最优不可能超过 LP 松弛值。本题整数最优 ≤ 41.25,即 ≤ 41。
- **分支(branch)**:挑一个取了小数的变量,劈成两个子问题。
  `x=3.75` → 分成 `x≤3` 和 `x≥4`(中间的 3<x<4 没有整数,合法地丢掉)。
- **定界 + 剪枝(prune)**:子问题若①不可行,②上界 ≤ 已找到的最好整数解,③本身已是整数解,
  就停止深挖。

### 跟着跑

```
根:LP松弛 z=41.25 (x=3.75,y=2.25)
│
├── 分支 x ≤ 3
│     解此子 LP → (3,3),z=39,已是整数 ✓  记下「目前最好 = 39」
│
└── 分支 x ≥ 4
      子 LP → (4,1.8),z=41,y 又是小数 → 继续对 y 分支
      ├── x≥4, y≥2 → 9·4+5·2=46>45 不可行,剪掉 ✂
      └── x≥4, y≤1 → 继续深挖,最终得整数 (5,0),z=40 ✓  更新「最好 = 40」
```

两支都探完:整数最优 = **(5,0),z=40**。

> 和 LP 松弛 41.25 比,整数最优 40 略低 —— 这个差距叫 **整数间隙(integrality gap)**。
> 求解器报告的 **optimality gap**(本仓库 [[../wiki/or-evaluation-metrics]] 提到的指标)就来自这里。

## 4. 0-1 变量:整数规划的建模利器

令变量只能取 **0 或 1**(binary),就能表达「**是/否**」决策,这是 IP 最强大的地方:

- **选址**:`y_i=1` 表示在第 i 地开仓,`0` 表示不开。
- **指派**:`x_ij=1` 表示把任务 i 派给人 j。
- **逻辑约束**:`x ≤ M·y` 表示「只有开了仓(y=1)才能从它发货(x>0)」(M 是大常数,叫 **big-M**)。

排班、背包、旅行商(TSP)、设施选址……几乎都靠 0-1 变量建模。

## 5. 一句话总结

- 整数要求让可行域变离散,顶点定理失效,**不能靠舍入**。
- **分支定界 = LP 松弛给上界 + 反复二分变量 + 剪枝**,是 MILP 求解器(Gurobi 等)的核心引擎。
- **0-1 变量**把「是否/选择/逻辑」这类离散决策装进线性模型,是建模表达力的关键。
