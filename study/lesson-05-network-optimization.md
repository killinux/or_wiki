# 第 5 课:网络优化 —— 最短路 / 最大流 / 指派

承接 [[lesson-04-integer-programming]]。一大类问题能画成「**点 + 边**」的网络,它们有比通用 LP/IP
**快得多**的专用算法。

## 0. 为什么网络问题特别「乖」

网络类问题的约束矩阵有个性质叫**全单位模(totally unimodular)**。它带来一个惊人结论:

> 🎯 即使不写整数约束,**LP 松弛的最优解自动就是整数**。所以网络问题不用分支定界,天生高效。

## 1. 最短路(Shortest Path)

给一张带权有向图,求两点间总权重最小的路径。例:

```
A→B=2  A→C=4  B→C=1  B→D=7  C→D=3   求 A→D 最短
```

**Dijkstra 算法**(每次扩展「当前已知最近」的点):
- 起点 `A=0`;由 A 更新 `B=2, C=4`。
- 取最近的 `B(2)`:更新 `C=min(4, 2+1)=3`,`D=2+7=9`。
- 取 `C(3)`:更新 `D=min(9, 3+3)=6`。
- 取 `D(6)`,结束。

✅ 最短路 **A→B→C→D,长度 6**。应用:导航、网络路由、最少换乘。

## 2. 最大流(Max Flow)

每条边有**容量上限**,问从源点 s 到汇点 t 最多能输送多少流量。例:

```
s→a=3  s→b=2  a→b=1  a→t=2  b→t=3
```
- `s→a→t` 推 2(受 a→t=2 限)
- `s→a→b→t` 再推 1(a→b=1)
- `s→b→t` 推 2
- 合计 **最大流 = 5**

> 🎯 **最大流最小割定理**:最大流量 = 最小割容量。这里把 s 单独切开,割 = `s→a + s→b = 5`,正好等于最大流。
> 应用:管道/电网调度、二分图匹配、图像分割。

## 3. 指派问题(Assignment)

把 n 个人分配给 n 件任务,每对有成本,求总成本最小的一一配对。可写成 0-1 规划:

```
min  Σ c_ij · x_ij
s.t. 每人恰好接 1 件任务:Σ_j x_ij = 1
     每件任务恰好 1 人做:Σ_i x_ij = 1
     x_ij ∈ {0,1}
```

虽然写了 0-1,但它是网络问题(全单位模)→ 直接解 LP 就得整数解。专用算法叫 **匈牙利算法**。
应用:排班、机器分配、订单派单。

## 4. 统一视角:最小费用流

最短路、最大流、指派、运输问题……都是**最小费用流(min cost flow)**的特例 —— 在带容量和单位费用的网络里,
以最小费用把规定流量从源送到汇。记住这个总框架,一类问题就串起来了。

## 一句话总结

- 能画成网络的问题(点+边)有**专用高效算法**,因为约束矩阵全单位模 → LP 解自动整数。
- 三件套:**最短路(Dijkstra)、最大流(最大流=最小割)、指派(匈牙利)**,都是**最小费用流**的特例。
