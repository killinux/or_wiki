# 第 7 课:非线性优化与启发式

承接 [[lesson-06-dynamic-programming]]。现实问题常常不是线性的,或大到求不出精确最优。
这一课讲两条出路:**非线性优化** 与 **启发式**。

## 1. 非线性规划(NLP)

目标或约束里出现 `x²`、`xy`、`√x`、`log x` 等,就是非线性规划。例:

```
min (x-3)² + (y-2)²     最小化到点 (3,2) 的距离平方
s.t. x + y ≤ 4
```

它比 LP 难,核心难点是:

> 🎯 **局部最优 ≠ 全局最优。** 非线性问题可能有很多「山谷」,算法容易掉进一个**局部**谷底,
> 误以为是全局最优。

## 2. 凸优化:非线性里的「好情形」

如果问题是**凸的**(目标是凸函数、可行域是凸集),局面立刻变好:

> 🎯 **凸问题里,任何局部最优都是全局最优。** 所以凸优化能可靠求解。

- 凸函数直觉:形状像碗(`x²`、`eˣ`),任意两点连线在函数图像上方。
- LP 是凸优化的特例;最小二乘、SVM、很多 ML 问题也是凸的。
- **判断问题凸不凸,是非线性建模的第一要务** —— 凸则可解,非凸则只能求近似。

求解思路多基于**梯度**(沿下坡方向迭代):梯度下降、牛顿法、内点法等。

## 3. 当问题「解不动」:启发式与元启发式

很多组合问题是 **NP-hard**(如大规模 TSP、排班):精确求最优要指数时间,实际不可行。
于是放弃「保证最优」,改求**快速得到足够好的解**——这就是**启发式(heuristic)**。

- **构造式启发式**:贪心地一步步搭一个解(如最近邻法走 TSP)。
- **局部搜索**:从一个解出发,不断做小改动(交换两条边)看能否改善。
- **元启发式(metaheuristic)**:跳出局部最优的通用策略——
  - **模拟退火**:以一定概率接受「变差」的一步,避免卡死在局部谷。
  - **遗传 / 演化算法**:维护一群解,通过「交叉 + 变异 + 选择」迭代进化。
  - **禁忌搜索、蚁群、粒子群**……

> 启发式**不保证最优**,但能在合理时间内给出实用解。代价是失去最优性证明。

## 一句话总结

- **非线性**的核心难点是局部最优陷阱;**凸优化**是好情形(局部=全局),判凸性是第一步。
- 问题大到 NP-hard 时,用**启发式/元启发式**(模拟退火、遗传算法)换取「快而够好」,放弃最优性保证。
- 这正是 LLM×OR 前沿的发力点:让 LLM 来生成/进化启发式、甚至直接当优化器。
