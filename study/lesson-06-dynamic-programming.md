# 第 6 课:动态规划(DP)—— 分阶段做决策

承接 [[lesson-05-network-optimization]]。当一个问题能拆成**一连串阶段**,每阶段做一个决策,
动态规划能避免重复计算、高效求最优。

## 1. 三个核心概念

- **阶段(stage)**:把决策过程切成有序的步骤(选第 1 件物品、第 2 件……)。
- **状态(state)**:走到某阶段时需要记住的信息(背包还剩多少容量)。
- **最优子结构 + Bellman 方程**:大问题的最优解由子问题的最优解拼成:

  > 🎯 `最优(当前状态) = 在所有可选决策里取最优{ 本步收益 + 最优(转移后的状态) }`

> DP 之所以快,是因为子问题**重叠**——同一个子问题只算一次、存表复用(记忆化)。这点和贪心不同:
> 贪心每步只顾眼前、不回头,DP 则系统比较所有可能。

## 2. 经典例子:0-1 背包

背包容量 **5**,4 件物品(重量, 价值):
```
物品1 (2,3)   物品2 (3,4)   物品3 (4,5)   物品4 (5,6)
```
每件要么拿(1)要么不拿(0),求不超重的最大总价值。

**DP 递推**:`dp[w]` = 容量 w 时的最大价值。逐件物品更新(容量从大到小,保证每件只用一次):

| 处理完 | dp[0] | dp[1] | dp[2] | dp[3] | dp[4] | dp[5] |
|--------|---|---|---|---|---|---|
| 初始 | 0 | 0 | 0 | 0 | 0 | 0 |
| +物品1(2,3) | 0 | 0 | 3 | 3 | 3 | 3 |
| +物品2(3,4) | 0 | 0 | 3 | 4 | 4 | **7** |
| +物品3(4,5) | 0 | 0 | 3 | 4 | 5 | 7 |
| +物品4(5,6) | 0 | 0 | 3 | 4 | 5 | **7** |

✅ 最大价值 **7** = 物品1+物品2(重 2+3=5,值 3+4=7)。

注意:单看「价值/重量」性价比贪心会选错 —— DP 才保证最优。

## 3. DP 能解什么

- **背包 / 资源分配**:有限预算分给多个项目。
- **最短路(无环图)**:其实就是 DP(按拓扑序递推)。
- **库存 / 设备更新 / 排程**:典型的多阶段决策。
- **维特比、最长公共子序列**:DP 在 CS 里也无处不在。

## 4. DP 的代价

状态空间可能爆炸(**维数灾难**):状态变量一多,表就指数级膨胀。这时退而用近似 DP 或启发式
(见 [[lesson-07-nonlinear-heuristics]])。

## 一句话总结

- DP = **分阶段 + 记状态 + Bellman 递推 + 子问题存表复用**,适合「一步步做决策」的问题。
- 它和贪心的区别:贪心只顾眼前不回头,DP 系统比较所有可能、保证最优。
- 代价是状态爆炸(维数灾难)。
